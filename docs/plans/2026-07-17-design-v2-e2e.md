**Goal:** One flow spec proving the whole v2 chain live: PDF → mapped design → colour-way beam → exact ledger drains.

**Architecture:** Spec §8 (e2e). Follows this repo's conventions: fixtures/test.ts `db` helper for oracle reads, `gotoAndExpect`, serial authed project. Runs after BE/FE tasks land.

- The spec must be runnable standalone via `npx playwright test tests/flows/design-v2.spec.ts` (dev servers stopped first — `reuseExistingServer: false`).
- Oracle reads go to `stock_ledger` (source-of-truth rule), never `/inventory`.
- Uses the copied fixture PDF, not a path outside the repo.
